# RISC-V Instruction Set Explorer

A Python command-line tool that parses the RISC-V instruction dictionary, cross-references extensions with the official ISA manual, and generates an instruction-overlap graph between extensions.

Implements all three tiers of the RISC-V Mentorship Coding Challenge.

---

## Features

| Tier | Description |
|------|-------------|
| **Tier 1** | Parse `instr_dict.json`, group instructions by extension, print summary table, list multi-extension instructions |
| **Tier 2** | Clone the RISC-V ISA manual, scan AsciiDoc sources, cross-reference extensions, report matched/JSON-only/manual-only |
| **Tier 3 (Bonus)** | Unit tests (pytest), extension overlap graph (networkx + matplotlib), this README |

```bash

```

## Running the Program

```bash

```


## Design Decisions

### Extension Name Normalisation (`utils.normalize_extension`)

The two data sources use different naming conventions:

| Source | Example |
|--------|---------|
| `instr_dict.json` | `rv_zba`, `rv64_zba`, `rv_i` |
| ISA manual AsciiDoc | `Zba`, `Zicsr`, `M`, `I` |

The normalisation function:
1. Strips `rv_` / `rv32_` / `rv64_` / `rv128_` prefixes (case-insensitive).
2. Collapses `z_ba` → `zba` style variants.
3. Capitalises single-letter extensions to uppercase (`m` → `M`).
4. Title-cases multi-letter extensions (`zba` → `Zba`, `svinval` → `Svinval`).

### ISA Manual Scanning

The AsciiDoc files are scanned with **original-case text** (not lowercased) so that:
- Single-letter patterns `[MAFDQCVBHJTP]` can distinguish extension letters from common English words.
- Mixed-case Z/S extension names (`Zba`, `Zicsr`) are matched directly.

Four regex patterns cover the main reference styles used in the manual:
- `\b(Z[a-zA-Z][a-z0-9]*)\b` — Z-extensions
- `\b(S[a-zA-Z][a-z0-9]*)\b` — S-extensions
- `\b([MAFDQCVBHJTP])\b` — single-letter base extensions
- `ext:name` and `[[ext:name]]` — AsciiDoc cross-reference anchors

### Overlap Graph

An edge is drawn between two extensions if they share **at least one instruction**. Node size is proportional to the number of instructions in the extension. The graph is saved as a PNG to `output/extension_overlap_graph.png` using a non-interactive matplotlib backend (safe in CI/headless environments).

### Assumptions

- `instr_dict.json` is assumed to be pre-downloaded into `data/`. The program exits with a clear error message if it is missing.
- The ISA manual is cloned automatically on first run (shallow, `--depth 1`) and reused on subsequent runs.
- The manual scanning is intentionally broad — it will report false positives (e.g., single-letter words in prose). These inflate `manual_only` counts but do not affect `json_only` accuracy.
